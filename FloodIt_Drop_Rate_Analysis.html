mart_level_funnel — Schema & Field Reference

Field Type Vai trò Ý nghĩa
level_number DIMENSION X-Axis chính Số thứ tự level (1, 2, 3... n). Grain = 1 row per level. Đây là trục chính để vẽ funnel.
users_completed METRIC Volume indicator Số user unique đã vượt qua level đó. Giảm dần theo level — tạo hình dạng funnel tự nhiên.
total_users METRIC Baseline Tổng user base (constant). Dùng làm denominator cho completion_rate.
completion_rate METRIC Primary Y-Axis users_completed / total_users. Tỷ lệ % user base đã hoàn thành level. Funnel chính.
avg_attempts METRIC Difficulty proxy Trung bình số lần thử để vượt level. Cao = level khó. Spike = difficulty wall.
avg_best_score METRIC Skill indicator Điểm cao trung bình tại level. Phản ánh skill ceiling & design fairness.
avg_days_to_complete METRIC Pacing indicator Trung bình số ngày kể từ install → hoàn thành level. Đo content consumption speed.

Calculated Fields — Tạo thêm trong Looker Studio
Các field bạn cần tạo thêm (không có sẵn trong mart) để phân tích drop rate chính xác
1. drop_rate (Level-over-Level Drop)
completion_rate của level [N-1] − completion_rate của level [N]
Ý nghĩa: % user base bị mất giữa 2 level liên tiếp. Đây là metric quan trọng nhất trong toàn bộ analysis. Dương = user drop, âm = không thể (funnel chỉ đi xuống). Trong Looker Studio: dùng LAG() window function hoặc tính trong BigQuery view mới.
2. drop_rate_pct (Drop Rate as % of previous level)
(completion_rate[N-1] − completion_rate[N]) / completion_rate[N-1] × 100
Ý nghĩa: % user bị mất tương đối so với level trước. Khác biệt: drop_rate là absolute, drop_rate_pct là relative. Level 80→81 drop 2% absolute có thể = 50% relative nếu chỉ còn 4% user.
3. difficulty_index (Composite Difficulty Score)
avg_attempts × (1 − completion_rate) × 100
Ý nghĩa: Kết hợp attempts + drop thành 1 chỉ số. Cao = level vừa khó vừa nhiều người bỏ. Dùng để rank "worst offender levels".
4. pacing_gap (Thời gian giữa 2 levels)
avg_days_to_complete[N] − avg_days_to_complete[N-1]
Ý nghĩa: Mất bao nhiêu ngày từ level trước sang level sau. Spike = pacing bottleneck — user bị kẹt quá lâu tại đó.

Cách đọc kết quả
Ưu tiên fix: góc trên-trái (Frustration Wall) — đây là levels đang đuổi user đi tích cực nhất.
Góc dưới-trái (Instant Quit) cũng cần fix nhưng khác cách: không phải nerf difficulty, mà cần redesign level layout/tutorial.
Levels ở góc trên-phải (Good Challenge): GIỮNGUYÊN — đây là điểm highlight của game.
Levels ở góc dưới-phải (Too Easy): có thể tăng nhẹ difficulty nếu cần kéo dài playtime.

Root-Cause Analysis Framework
Sau khi xác định level có drop cao, dùng framework sau để chẩn đoán nguyên nhân gốc rễ

Root Cause 1: Difficulty Spike

Triệu chứng: Drop rate cao + avg_attempts cao + avg_best_score thấp tại cùng level

Nguyên nhân: Level quá khó so với level trước — jump in difficulty không smooth. Board layout phức tạp đột ngột, số màu tăng, moves giảm quá nhiều.

Signal: avg_attempts > 2× median
AND drop_rate > AVG × 2
AND avg_best_score dip

Root Cause 2: RNG / Unfair Design

Triệu chứng: Drop rate cao + avg_attempts vừa + score distribution rất hẹp (mọi user score gần nhau nhưng vẫn fail)

Nguyên nhân: Level phụ thuộc RNG quá cao — board random không có đủ "winnable" configurations. User skill không giúp ích → frustrating.

Signal: drop_rate cao
BUT avg_attempts moderate (1.5-3x)
AND score spread hẹp (Q75-Q25 nhỏ)

Root Cause 3: Confusion / UX Gap

Triệu chứng: Drop rate cao + avg_attempts RẤT THẤP (user quit mà không thử) + level thường ở giai đoạn sớm (1-15)

Nguyên nhân: Level giới thiệu mechanic mới nhưng không có tutorial. User không hiểu phải làm gì → bỏ ngay. Hoặc UI/board confusing.

Signal: drop_rate cao
BUT avg_attempts < median
AND level_number < 15-20

Root Cause 4: Content Fatigue

Triệu chứng: Drop rate tăng dần đều ở mid-to-late game + avg_attempts normal + pacing_gap tăng

Nguyên nhân: Gameplay lặp lại — không có mechanic mới, visual variety giảm, reward cycle không đủ hấp dẫn. User hết motivation chứ không phải hết skill.

Signal: drop_rate tăng dần (không spike)
AND avg_attempts stable
AND avg_days_to_complete gap tăng

Root Cause 5: Paywall Friction

Triệu chứng: Drop spike tại level specific = level đầu tiên yêu cầu IAP/extra moves/lives. Chỉ non-payer drop, payer pass through.

Nguyên nhân: Paywall đặt quá sớm hoặc quá aggressive. User chưa đủ invested để trả tiền → bỏ game thay vì mua.

Signal: drop_rate spike tại 1 level cụ thể
AND level trùng với IAP trigger point
Cross-ref: mart_player_segments

Root Cause 6: Technical Issue

Triệu chứng: Drop spike đột ngột tại 1 level + không correlate với difficulty metrics + có thể platform-specific

Nguyên nhân: Bug, crash, loading issue tại level cụ thể. Thường xảy ra sau update version. Need cross-check với crash logs.

Signal: drop_rate spike
BUT avg_attempts normal & score normal
AND possibly platform-specific

Quick Reference — Chart Reading Flow
Step Chart Bạn đang tìm gì? Nếu thấy anomaly → Tiếp theo
1 C1 — Level Completion Funnel Hình dạng funnel tổng thể. Có "cliff" rõ ràng không? Thấy cliff → Zoom vào C2 để xem drop cụ thể
2 C2 — Drop Rate Spike Detector Level NÀO drop nhiều nhất? Bar nào vượt reference line? Ghi lại level numbers → Tra C3 để phân loại nguyên nhân
3 C3 — Difficulty Quadrant Level drop thuộc quadrant nào? Frustration vs Confusion vs Quit? Theo quadrant → Áp root cause tương ứng
4 C4 — Multi-metric Overlay Drop có correlate với attempts & score không? Pattern A, B, hay C? Confirm root cause → Chọn action từ Action Plan
5 C5 — Pacing Curve User bị stuck bao lâu? Có pacing bottleneck không? Bottleneck → Thêm content/events tại vùng đó
6 C6 — Problem Levels Table Top N worst levels, ranked. Export → gửi GD team GD team tạo Jira tickets cho từng level cần fix
