Theme 1 — Level Drop & Difficulty Walls
Focus: Tìm level nào đang chặn user, tại sao, và đề xuất điều chỉnh cho Game Design team
1.1
Level nào có completion rate drop mạnh nhất (>15% so với level trước)?
Khi completion rate giảm đột ngột giữa 2 level liên tiếp, đó là difficulty wall — nơi user bỏ game nhiều nhất. Giống Bike Stunt v1.2.5→1.2.9 vẫn chưa cải thiện, cần xác định chính xác vị trí drop.
Data Source & Fields
mart_level_funnel
Fields: level_number completion_rate users_completed
Tính: completion_rate[n] - completion_rate[n-1] → sort desc
Insight kỳ vọng
Danh sách top 5-10 "cliff levels" — vd: Level 12→13 drop 22%, Level 31→32 drop 18%...
Cross-check avg_attempts để xác nhận user đang struggle hay chỉ bỏ ngang
Action Brief
Game Design
→ Nerf difficulty tại các cliff levels (giảm số màu, tăng moves)
→ Thêm hint/tutorial popup tại level drop cao nhất
→ Cân nhắc thêm "bonus level" dễ trước cliff để giữ momentum
1.2
Level nào có avg_attempts cao bất thường (>2x median) nhưng users vẫn cố gắng vượt qua?
Phân biệt 2 loại level khó: (A) khó nhưng user persist → "good challenge", (B) khó và user bỏ luôn → "frustration wall". Loại A giữ nguyên, loại B cần nerf.
Data Source & Fields
mart_level_funnel
Fields: avg_attempts completion_rate users_completed
Matrix: attempts (cao/thấp) × completion (cao/thấp)
Insight kỳ vọng
Scatter plot 4 quadrants:
• High attempts + High completion = Good challenge ✅
• High attempts + Low completion = Frustration wall ❌
• Low attempts + Low completion = User gives up immediately ❌❌
Action Brief
Game Design
→ Nerf "Frustration wall" levels: giảm board size, tăng max moves
→ "Give up immediately" levels: review level design root cause — quá confusing hay quá hard?
→ Keep "Good challenge" levels — chúng tạo sense of achievement
1.3
User mất bao lâu để đến các level milestone (10, 20, 50, 100)? Có bottleneck timing nào không?
Nếu avg_days_to_complete tăng đột biến tại một level range nhất định, user đang bị "stuck" quá lâu → mất hứng thú. Pacing quá chậm = churn risk, quá nhanh = hết content sớm.
Data Source & Fields
mart_level_funnel
Fields: level_number avg_days_to_complete
Plot: X=level, Y=avg_days → tìm inflection points
Insight kỳ vọng
Pacing curve: level 1-10 trong 1 ngày, level 10-20 trong 3 ngày... Nếu level 30-40 mất 15+ ngày → content pacing vấn đề
Action Brief
Game Design Product
→ Rebalance pacing curve: ideal = logarithmic growth
→ Thêm daily rewards / challenges tại vùng bottleneck để duy trì engagement
→ Xem xét thêm content giữa 2 milestone nếu gap quá lớn
1.4
Score distribution tại các level drop có gì bất thường? User fail vì thiếu skill hay vì level design không công bằng?
Nếu score distribution rất hẹp (mọi user đều score gần nhau nhưng vẫn fail) → level design vấn đề, không phải skill issue. Nếu distribution rộng → skill gap, có thể giải quyết bằng dynamic difficulty.
Data Source & Fields
mart_level_funnel
Fields: avg_best_score kết hợp fact_levels cho score distribution
So sánh score spread tại cliff levels vs normal levels
Insight kỳ vọng
Narrow score spread + high fail = level cần redesign (RNG quá cao, board layout unfair)
Wide score spread = normal skill-based difficulty
Action Brief
Game Design
→ Narrow spread levels: regenerate board layouts, reduce RNG factor
→ Wide spread levels: add adaptive hint system for lower-skill players
→ Cân nhắc dynamic difficulty adjustment (DDA) cho nhóm at_risk

Theme 2 — Retention Drop Diagnostics
Focus: Xác định user rời game ở giai đoạn nào, tại sao, và action cho cả Game Design + Marketing
2.1
D1 retention đang ở bao nhiêu? Có đang dưới benchmark 35-45% (casual game) không? Trend tăng hay giảm?
D1 retention phản ánh first impression — tutorial, UX onboarding, loading time. Nếu D1 < 30% = game mất user ngay ngày đầu, fix onboarding trước khi đổ tiền UA.
Data Source & Fields
mart_retention_cohorts
Fields: d1_retention cohort_date cohort_size
Trend: AVG(d1_retention) theo weekly cohort
Insight kỳ vọng
D1 trend line: stable, improving, or declining?
So sánh D1 theo platform — Android vs iOS thường khác nhau
So sánh D1 theo traffic_source — paid users thường D1 thấp hơn organic
Action Brief
Game Design Marketing
→ D1 thấp: Review tutorial flow, FTUE (First Time User Experience), loading time
→ D1 thấp chỉ ở paid traffic: Creative mismatch → Marketing đổi ad creative
→ D1 thấp ở 1 platform: Platform-specific bugs → QA team
2.2
Retention drop lớn nhất xảy ra giữa Dx nào → Dy nào? (D1→D3, D3→D7, D7→D14, D14→D30?)
Mỗi giai đoạn drop tương ứng một nguyên nhân khác nhau: D1→D3 = core loop yếu, D3→D7 = thiếu content depth, D7→D14 = thiếu social/meta, D14→D30 = endgame boring.
Data Source & Fields
mart_retention_cohorts
Tính: d3_retention - d1_retention, d7 - d3, d14 - d7, d30 - d14
Tìm giai đoạn nào drop % lớn nhất
Insight kỳ vọng
Nếu D1→D3 drop 50%+ = core gameplay loop chưa hook được user
Nếu D7→D14 drop nhiều nhất = user hết content/progression trong tuần đầu
Pattern này chỉ ra chính xác GĐ nào cần intervention
Action Brief
Game Design Product
→ D1→D3: Improve core loop — thêm biến thể gameplay, reward cycle ngắn hơn
→ D3→D7: Thêm daily quest, streak rewards, new mechanic unlock
→ D7→D14: Social features, leaderboard, guild/friends
→ D14→D30: Endgame content, seasonal events, prestige system
2.3
Nhóm player nào đang chiếm tỉ lệ lớn nhất? Tỉ lệ churned + at_risk hiện tại là bao nhiêu?
Nếu churned + at_risk > 50% user base → game đang trong giai đoạn suy thoái. Cần win-back campaigns ngay. Nếu engaged_free lớn → cơ hội convert to payer.
Data Source & Fields
mart_player_segments
Aggregate: COUNT(*) GROUP BY player_segment
Also: AVG(active_days) AVG(max_level) per segment
Insight kỳ vọng
Segment % pie: casual (lớn nhất?) → at_risk → churned → engaged_free → minnow → dolphin → whale
at_risk segment profile: avg level bao nhiêu, chơi bao lâu rồi bỏ?
Action Brief
Marketing Product
→ at_risk: Push notification win-back, limited-time event
→ churned: Re-engagement email/ad campaign with new content hook
→ engaged_free: Soft paywall experiment, first-purchase discount
→ casual: Optimize FTUE to convert → engaged_free
2.4
Retention khác nhau thế nào giữa các traffic source? Nguồn UA nào mang user chất lượng cao nhất?
Paid users thường D1 thấp hơn organic. Nếu một nguồn paid traffic có D7 < 5% → đang đốt tiền vô ích. Cần reallocate budget sang source có retention tốt hơn.
Data Source & Fields
mart_retention_cohorts
Breakdown: traffic_source
Compare: d1_retention d7_retention d30_retention per source
Insight kỳ vọng
Ranking traffic sources by D7 retention quality
Source có volume cao + retention thấp = wasted spend
Source có volume thấp + retention cao = scale opportunity
Action Brief
Marketing
→ Shift UA budget: giảm spend ở source low-retention, tăng ở high-retention
→ Low-retention source: review targeting (wrong audience?) hoặc creative (misleading?)
→ Organic retention cao nhất? → Invest in ASO & content marketing

Theme 3 — Monetization & Ad Optimization
Focus: Tìm chỗ revenue leak, ad placement gây drop cao nhưng revenue thấp, và cơ hội monetize mới (giống case Bike Stunt)
3.1
Revenue mix hiện tại (IAP vs Ad vs Virtual) thế nào? Nguồn nào đang dominate và có sustainable không?
Nếu >80% revenue từ ads → rủi ro cao khi ad rates giảm. Nếu IAP quá thấp → monetization design chưa tốt. Cần balance revenue portfolio giống đa dạng hóa đầu tư.
Data Source & Fields
mart_revenue_daily
Aggregate: SUM(total_revenue) GROUP BY revenue_type
Trend: revenue_type breakdown over time
Insight kỳ vọng
Revenue pie chart: IAP chiếm bao nhiêu %? Ad chiếm bao nhiêu?
Trend: nguồn nào đang tăng/giảm theo thời gian?
Benchmark: casual game thường 60-70% ad, 30-40% IAP
Action Brief
Monetization Game Design
→ IAP quá thấp: Review IAP catalog — giá, value proposition, placement timing
→ Ad quá dominant: Diversify — thêm IAP bundles, battle pass, cosmetics
→ Virtual currency imbalance: Tune economy — sink vs faucet ratio
3.2
Ad impressions xảy ra ở level nào? Level nào có ad frequency cao nhưng retention drop cũng cao → ads đang đuổi user đi?
Giống case Bike Stunt: vị trí ads nào đang gây drop cao mà doanh thu mang lại không đáng? Cần tìm "toxic ad placements" — nơi ads hurt retention nhiều hơn revenue chúng mang lại.
Data Source & Fields
mart_level_funnel + mart_revenue_daily (cross-ref fact_levels + fact_revenue)
Cần join: ad events by level → compare ad frequency vs completion_rate drop
Nếu mart chưa có: query fact_events WHERE event_name='ad_impression' GROUP BY param_level
Insight kỳ vọng
Matrix: level × ad_impressions × completion_rate_drop
"Toxic zones": high ad frequency + high user drop + low revenue per impression
"Sweet spots": moderate ad frequency + stable retention + decent revenue
Action Brief
Monetization Game Design
→ Toxic zones: giảm ad frequency hoặc đổi sang rewarded ads (user opt-in)
→ Sweet spots: có thể tăng nhẹ ad frequency
→ Suggest kịch bản ads mới: rewarded video cho extra moves, interstitial chỉ sau level win (không phải fail)
3.3
Payer conversion rate hiện tại bao nhiêu? User chuyển sang trả tiền tại thời điểm nào trong lifecycle?
Nếu conversion < 2% → paywall chưa compelling. Biết user convert ở ngày thứ mấy sau install → optimize timing hiển thị offer.
Data Source & Fields
mart_daily_kpis: payer_conversion_rate trend
mart_player_segments: whale + dolphin + minnow → first purchase timing
Cross-ref: dim_users.install_date vs fact_revenue.event_date
Insight kỳ vọng
Conversion rate trend: improving or declining?
First purchase timing distribution: Day 0? Day 3? Day 7?
Conversion by segment: casual vs engaged_free → ai convert?
Action Brief
Monetization Marketing
→ Low conversion: Starter pack ($0.99) tại peak conversion day
→ Late conversion (>D7): Earlier soft IAP touchpoints cần thiết
→ engaged_free lớn nhưng không convert: A/B test IAP pricing & value
3.4
Whale users (top spenders) có profile gì chung? Level bao nhiêu, chơi bao lâu, từ market nào?
Whale chiếm phần lớn IAP revenue. Hiểu profile whale → optimize để tìm & nuôi thêm whales tương tự. Nếu whale chỉ đến từ 1-2 markets → opportunity ở các market khác.
Data Source & Fields
mart_player_segments WHERE player_segment='whale'
Profile: avg(max_level) avg(active_days) avg(total_engagement_minutes) geo_country
Insight kỳ vọng
Whale profile: level 50+, active 20+ days, platform X, from country Y
Compare with dolphin profile → what's the tipping point from dolphin → whale?
Action Brief
Marketing Monetization
→ Lookalike audience: Target user giống whale profile trên paid UA
→ Dolphin → Whale: VIP program, exclusive content, progressive discounts
→ Under-indexed markets with whale potential: localize + UA push

Theme 4 — Session & Engagement Health
Focus: User chơi có "sâu" không? Session chất lượng hay chỉ open-close? Dấu hiệu engagement giảm sớm.
4.1
Tỉ lệ sessions dưới 1 phút (sessions_under_1min) chiếm bao nhiêu %? Trend tăng hay giảm?
Session < 1 phút = user mở game rồi đóng ngay (bounce). Nếu tỉ lệ này > 30% → nghiêm trọng: loading quá lâu, crash, hoặc push notification không lead đến content hay.
Data Source & Fields
mart_session_stats
Calculated: sessions_under_1min / total_sessions
Trend: weekly average
Insight kỳ vọng
Bounce session % trend — stable ≈ structural issue, increasing = regression/bug
Breakdown by platform → crash issue trên specific device?
Action Brief
Game Design Product
→ High bounce: Check loading time, crash logs, first screen appeal
→ Platform-specific: Device compatibility issue → QA
→ Push notification bounce: Improve notification → content pathway
4.2
Engagement ratio (engagement_sec / duration_sec) bao nhiêu? User idle bao nhiêu % thời gian trong session?
Ratio < 50% = user dành nửa thời gian không tương tác (xem ads? waiting? confused?). Đây là proxy cho UX quality — cao = flow tốt, thấp = friction nhiều.
Data Source & Fields
mart_session_stats hoặc mart_daily_kpis
Calc: avg_engagement_sec / avg_duration_sec
Trend: daily
Insight kỳ vọng
Engagement ratio 60-80% = healthy
< 50% = too much idle time → investigate: long loading, forced ads, confusing menus
Drop over time = UX degradation
Action Brief
Game Design Product
→ Low ratio: Reduce loading screens, minimize forced wait times
→ Declining ratio: Audit recent version updates for added friction
→ Compare by platform — iOS vs Android UX parity check
4.3
Sessions per DAU có tương quan với retention không? User chơi nhiều session/ngày có retain tốt hơn?
Nếu high-frequency users cũng là high-retention users → sessions_per_dau là leading indicator. Có thể dùng để early detect churn khi sessions/dau giảm.
Data Source & Fields
mart_daily_kpis: sessions_per_dau trend
Cross: mart_player_segments → avg sessions per segment
Insight kỳ vọng
engaged_free: sessions/day ~3-4x vs casual: ~1.2x
Tìm "magic number" session frequency → retention threshold
Action Brief
Game Design Product
→ Tìm magic number (vd: 3 sessions/day) → design push/reward tại moment 2 sessions để đẩy lên 3
→ Energy system / daily login rewards to drive repeat sessions

Theme 5 — Geo & UA Effectiveness
Focus: Market nào đáng đầu tư? Country nào có user quality cao nhất? Budget allocation cho Marketing
5.1
Top 10 countries theo revenue — có khớp với top 10 theo user count không? Country nào high-users nhưng low-revenue (monetization gap)?
Country có nhiều user nhưng payer_rate thấp = chưa monetize tốt ở market đó. Có thể do pricing không phù hợp purchasing power, hoặc IAP catalog chưa localized.
Data Source & Fields
mart_geo_stats
Rank by: total_users vs total_revenue
Key: payer_rate gap between top-user vs top-revenue countries
Insight kỳ vọng
Mismatch ranking = opportunity: vd India #1 users nhưng #20 revenue → pricing issue
US/Japan/Korea thường ARPU cao → đáng UA invest
Action Brief
Marketing Monetization
→ High-user low-revenue: Regional pricing (India ₹, SEA lower price points)
→ High-ARPU countries: Increase UA spend, localize ads
→ Consider geo-specific IAP bundles & ad mediation
5.2
Engagement quality (avg_active_days, avg_level, avg_engagement_min) khác nhau thế nào giữa các geo regions?
Không phải market có nhiều user nhất là tốt nhất. Market có avg_active_days cao + avg_level cao = user chất lượng thực sự. Đây là leading indicator cho LTV.
Data Source & Fields
mart_geo_stats
Fields: avg_active_days avg_level avg_engagement_min
Rank countries by composite quality score
Insight kỳ vọng
Quality ranking vs volume ranking → khác nhau = targeting opportunity
Continent-level patterns: Americas vs Europe vs Asia engagement styles
Action Brief
Marketing
→ High quality + low volume = under-tapped market → UA campaign
→ Low quality + high volume = improve FTUE for that market, or adjust targeting
→ Platform preference by geo → focus Android UA in emerging markets

Theme 6 — Product Health Pulse (Daily/Weekly Check)
Focus: Câu hỏi PO nên check hàng ngày/tuần để phát hiện sớm anomaly — "early warning system"
6.1
DAU hôm nay so với 7-day moving average thế nào? Có anomaly (drop >15% hoặc spike >25%) không?
Morning health check #1. DAU spike = viral moment hoặc UA campaign hit. DAU drop = server issue, bad update, hoặc market event. Cần phát hiện trong vài giờ, không phải vài ngày.
Data Source & Fields
mart_daily_kpis
dau vs 7-day rolling AVG
Alert threshold: ±15%
Check Frequency
Daily — mỗi sáng
Set up Looker Studio email schedule hoặc alert
Escalation
Product Data
→ Drop: Check crash reports, server status, recent deploy
→ Spike: Identify source → capitalize if organic, validate if paid
6.2
Revenue hôm nay có on-track so với weekly/monthly target không?
Morning health check #2. Revenue lagging = cần intervention (sale event, new offer). Revenue exceeding = understand why → replicate.
Data Source & Fields
mart_daily_kpis: total_revenue
mart_revenue_daily: breakdown by type
Cumulative sum vs target line
Check Frequency
Daily — revenue tracking
Weekly — pace vs monthly target
Escalation
Monetization Marketing
→ Behind target: Flash sale, limited offer, ad placement review
→ Ahead: Analyze driver → scale what's working
6.3
Stickiness (DAU/MAU) trend trong 4 tuần gần nhất — tăng, giảm, hay flat?
Weekly health check. Stickiness giảm dần = game đang mất grip — user vẫn trong MAU nhưng không active daily. Đây là leading indicator cho churn wave sắp đến.
Data Source & Fields
mart_daily_kpis: stickiness
7-day rolling average
Benchmark: casual game 15-25%
Check Frequency
Weekly
Compare week-over-week
Escalation
Game Design Product
→ Declining: Review daily hooks — daily rewards, notifications, live events
→ Below 15%: Urgent — core loop may not be compelling enough
→ Stable/growing: Healthy — maintain current strategy
6.4
New install volume tuần này so với tuần trước? Cost per Install có đang tăng không?
Weekly UA check. Install giảm + DAU giảm = acquisition problem. Install stable + DAU giảm = retention problem. Phân biệt gốc rễ vấn đề để assign đúng team.
Data Source & Fields
mart_daily_kpis: new_users weekly sum
CPI data từ ad network (ngoài mart — cần external source)
Check Frequency
Weekly
Monday morning review
Escalation
Marketing
→ Install drop + CPI stable: Market saturation → new geo/creative
→ Install stable + CPI rising: Competition → differentiate creative
→ Organic ratio check: healthy if organic > 40% of total installs
