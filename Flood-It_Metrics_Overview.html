1. User Acquisition & Growth
mart_daily_kpis
DAU (Daily Active Users)
COUNT(DISTINCT user_pseudo_id) per day
Số user unique hoạt động mỗi ngày. North star metric phản ánh sức khỏe game hàng ngày.
North Star
MAU (Monthly Active Users)
COUNT(DISTINCT user) trong 30-day rolling
User unique trong 30 ngày gần nhất (rolling window). Cho thấy tổng quy mô active player base.
North Star
Stickiness (DAU/MAU)
DAU / MAU
Tỷ lệ user quay lại hàng ngày so với tổng MAU. Benchmark game casual: 15-25%. Càng cao = game càng "gây nghiện".
Engagement
New Users (Daily Installs)
COUNT(*) WHERE install_date = today
Số user mới cài đặt mỗi ngày. Tracking tốc độ tăng trưởng organic & paid.
Acquisition
🔁
2. Retention & Churn
mart_retention_cohorts mart_player_segments
D1 Retention
d1_users / cohort_size
% user quay lại sau 1 ngày kể từ install. Benchmark casual game: 35-45%. Phản ánh first impression & onboarding quality.
Critical
D7 Retention
d7_users / cohort_size
% user còn lại sau 1 tuần. Benchmark: 15-20%. Đo lường core loop có đủ hấp dẫn để giữ player không.
Critical
D14 / D30 Retention
d14_users / cohort_size, d30_users / cohort_size
Long-term retention. D30 benchmark casual: 5-8%. Nếu thấp hơn, cần review content depth & progression system.
Long-term health
Player Segments
RFM-based: whale, dolphin, minnow, engaged_free, casual, at_risk, churned
Phân loại user theo Recency-Frequency-Monetary. Quan trọng để biết tỉ lệ phân bổ và target từng nhóm khác nhau.
Segmentation
Cohort Size
COUNT(DISTINCT user) WHERE days_since_install = 0
Kích thước cohort mỗi ngày, dùng cùng với retention % để biết số tuyệt đối user còn lại.
Supporting
D3 Retention
d3_users / cohort_size
Checkpoint giữa D1 và D7, giúp phát hiện drop-off sớm nếu player cảm thấy game lặp lại.
Diagnostic
💰
3. Revenue & Monetization
mart_daily_kpis mart_revenue_daily mart_player_segments
Total Revenue (Daily)
SUM(revenue_amount) per day
Tổng doanh thu hàng ngày (USD). Breakdown theo revenue_type: IAP, ad, virtual currency.
North Star
ARPU
total_revenue / DAU
Average Revenue Per User. Đo hiệu quả monetize trên toàn bộ user base (kể cả free users).
Unit economics
ARPPU
total_revenue / paying_users
Average Revenue Per Paying User. Phản ánh mức chi trung bình của nhóm user đã trả tiền.
Monetization
Payer Conversion Rate
paying_users / DAU
% user chuyển sang trả tiền. Benchmark F2P casual: 2-5%. Metric then chốt đánh giá paywall design.
Conversion
Revenue by Type
SUM(revenue) GROUP BY revenue_type
Phân loại: IAP vs Ad revenue vs Virtual currency. Cho biết revenue mix và nguồn doanh thu chính.
Revenue mix
Avg Transaction Value
AVG(revenue_amount)
Giá trị trung bình mỗi giao dịch. Kết hợp transaction_count để hiểu hành vi chi tiêu.
Pricing
Segment Revenue
SUM(total_revenue) GROUP BY player_segment
Revenue theo nhóm player (whale/dolphin/minnow). Whale thường đóng góp 50%+ tổng doanh thu.
Segmentation
Unique Payers (Daily)
COUNT(DISTINCT user) WHERE revenue > 0
Số lượng user trả tiền mỗi ngày. Track xu hướng tăng/giảm để đo sức khỏe monetization.
Paying base
⏱
4. Session & Engagement Quality
mart_daily_kpis mart_session_stats
Avg Session Duration
AVG(session_duration_sec)
Thời lượng trung bình 1 session. Casual game benchmark: 3-8 phút. Quá ngắn = thiếu content, quá dài = burnout risk.
Core engagement
Sessions per DAU
total_sessions / DAU
Trung bình số lần mở game/ngày/user. Cao = game có pull-back loops tốt (notifications, daily rewards...).
Frequency
Avg Engagement Time
AVG(total_engagement_sec)
Thời gian user thực sự tương tác (không tính idle). Chính xác hơn session duration.
True engagement
Avg Levels per Session
AVG(levels_completed) per session
Số level hoàn thành/session. Indicator quan trọng cho content consumption pace.
Progression
Session Length Distribution
<1m | 1-5m | 5-15m | 15m+
Phân bổ 4 nhóm session length. Nếu đa số <1 phút = vấn đề load time hoặc onboarding fail.
Distribution
Median Session Duration
APPROX_QUANTILES(duration, 100)[50]
Median chính xác hơn AVG vì loại bỏ outliers (session quá dài do AFK). So sánh với AVG để phát hiện skew.
Diagnostic
Avg Events per Session
AVG(events_count)
Mức độ tương tác (tap, swipe, action) trong mỗi session. Proxy cho session richness.
Depth
Total Sessions (Daily)
COUNT(*) per day
Tổng số session mỗi ngày. Kết hợp DAU để tính sessions per user.
Volume
🎮
5. Level Progression & Game Design
mart_level_funnel
Level Completion Rate
users_completed / total_users per level
% tổng user base đã vượt qua từng level. Chart dạng funnel — chỗ nào drop mạnh = level quá khó hoặc thiếu motivation.
Game design
Avg Attempts per Level
AVG(attempts)
Trung bình số lần thử mỗi level. Spike đột biến = difficulty wall. Target: tăng dần đều, không có cliff.
Difficulty curve
Avg Best Score per Level
AVG(best_score)
Điểm cao trung bình mỗi level. Kết hợp với attempts để đánh giá skill ceiling & reward feeling.
Skill curve
Avg Days to Complete
AVG(first_completed_date - install_date)
Trung bình bao nhiêu ngày sau install user mới vượt level đó. Đo pacing & content consumption speed.
Pacing
Users Completed (per level)
COUNT(DISTINCT user) per level_number
Số lượng tuyệt đối user vượt qua mỗi level. Kết hợp completion_rate để ra bức tranh đầy đủ.
Volume
🌍
6. Geographic & Market Distribution
mart_geo_stats
Users by Country
COUNT(DISTINCT user) GROUP BY geo_country
Top markets theo user count. Quyết định localization priority & UA spend allocation.
Market sizing
Payer Rate by Country
payers / total_users per country
Tỉ lệ user trả tiền theo quốc gia. Giúp identify high-value markets cho monetization focus.
Monetization
Revenue by Country
SUM(total_revenue) per country
Tổng doanh thu theo quốc gia. Cross-reference với user count để tìm high-ARPU markets.
Revenue
Avg Engagement by Geo
AVG(engagement_min), AVG(active_days), AVG(level)
So sánh engagement depth giữa các thị trường: active days, level reached, playtime. Phát hiện thị trường có user chất lượng cao.
Quality
